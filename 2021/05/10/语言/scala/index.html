1.概述

1.1 Scala和Java的关系

1.2 为什么学习scala

1）很多大数据的引擎框架都是基于scala进行开发的，比如spark，flink

2）在大数据领域中，很多场景都是数据的分析处理，scala的函数式编程开发效率高，而且直观易于理解

1.3 如何在不同的场景下区分使用java还是scala呢

java：如何强调的是业务处理之间的逻辑关系，比如电商系统的用户登录，订单支付等等，那就用java，强调的是对象之间的关系，因为本身java就是面向对象的语言

scala：而如果强调的是数据之间的转换和分布式统计，那就采用scala，scala的函数式编程可以让处理的过程变得简单，代码开发变得更加简洁

2.Hello world解析

java：

java Helloworld 执行的时候会调用类的静态方法main，是通过类来调用的，而非创建对象

scala

scala是完全面向对象的语言，所以摒弃了和面向对象无关的语法内容，也就不存在通过类来调用静态方法的语法

如果scala中的类想要通过名称来访问方法，那么必须创建一个使用object关键字声明的特殊的对象（伴生对象），这个对象的名称

和类名保持一致，所以可以通过对象访问方法。

用这种方式来模拟java的静态语法

我们一般将使用object关键字声明的类称之为伴生对象，是单例的。

def :

​ 英文单词define的缩写：表示声明一个方法

main:

​ 表示scala语言的主方法

args：Array[String]

Unit:

​ java中void关键字不是面向对象的语法内容，所以在scala中是没有的，所以在scala中提供Unit类型来代替void，模拟无返回值的场合

=:

​ =表示将代码逻辑进行封装赋值给方法，变量和方法都是用=赋值，体现了访问的统一性

System.out.println():

​ scala语言是基于java开发的，所以java的类库在scala程序中可以直接使用，但是在scala中进行了简化（println()），提高开发效率

分号:

​ java为了明确代码的关系，所以采用分号进行结尾，表示代码逻辑终止。为了查看方便，一般不会将多行代码写在一行中

​ scala为了简化开发，如果一行代码中只有一段逻辑，那么结尾的分号可省略，使用换行来分隔

​ 如果将多个逻辑写在一行中，那么必须使用分号分隔

如果函数体返回值类型明确为Unit, 那么函数体中即使有return关键字也不起作用

6.面向对象

6.1 属性

在类中声明属性，其实并不是声明属性，而是声明方法。在编译时，编译器会将类的属性自动编译为类的私有属性；编译的同时，编译器会给这个属性同时编译成两个公共的set/get方法。在直接访问属性是，其实是调用的get方法，直接赋值时等同于调用set方法

如果属性使用val声明，那么属性值一旦初始化后无法改变。编译器在编译时，val的属性会自动添加final关键字，而且没有提供对应的set方法

如果属性声明时使用private关键字，那么编译时的属性和set,get方法都是私有的

scala默认生成的set 和get方法并不是以set和get开头的

如果想要scala中类的属性方法符合Bean的规范，需要增加特殊的注解（@BeanProperty），添加这个注解的属性，编译时默认会增加对应的get和set方法

8.模式匹配

scala中的模式匹配类似于java中的switch语法，但是scala从语法中补充了更多的功能

8.1 基本语法

模式匹配语法中，采用match关键字声明，每个分支采用case关键字进行声明，当需要匹配时，会从第一个case分支开始，如果匹配成功，那么执行对应的逻辑代码，如果匹配不成功，继续执行下一个分支进行判断。如果所有case都不匹配，那么会执行case _分支，类似于Java中default语句。

8.2.7 样例类

l 样例类就是使用case关键字声明的类

l 样例类仍然是类，和普通类相比，只是其自动生成了伴生对象，并且伴生对象中自动提供了一些常用的方法，如apply、unapply、toString、equals、hashCode和copy。

l 样例类是为模式匹配而优化的类，因为其默认提供了unapply方法，因此，样例类可以直接使用模式匹配，而无需自己实现unapply方法。

l 构造器中的每一个参数都成为val，除非它被显式地声明为var（不建议这样做）

10.隐式转换

10.1 简介

Scala中在程序编译错误时，可以通过内置的类型转换机制进行二次编译，尝试将本身错误的代码通过类型转换后编译通过。慢慢地，这也形成了一种扩展功能的转换机制

简单地说，隐式转换是将值的类型自动从一种类型转换为另外一种类型

这个转换的操作，开发者是看不到的，所以称之为隐式转换

注意：在相同的作用域中不能存在相同的转换规则，不然编译器就会不知道选择哪个无法完成隐式转换

隐式转换主要用于扩展功能

10.4 隐式类

单独声明需要被转换的目标类和单独声明隐式转换的方法比较麻烦，所以在scala2.10后提供了隐式类

隐式类用implicit进行修饰，有以下特点：

1）其所带的构造参数有且只有一个

2）隐式类必须定义在“类”或“伴生对象”或“包对象”里，即隐式类不能是顶层类

10.5 隐式机制

隐式转换规则：

​ 1）隐式转换需要在当前作用域中或者跟当前作用域有关联的域中可以找到

可以在当前作用域，当前作用域的父类，特质，包对象中都可以查找因式转换，如果在指定的作用域中查找不到需要用到的隐式转换，可以通过import的方式进行导入

10.6 功能扩展

隐式转换一般用于功能扩展

扩展功能主要有以下两种方式

特质混入，隐式转换。那如何选择呢？

1）trait：interface&abstract 里面一般封装的是通用的功能，如果是扩展一个项目中通用的功能那就用特质混入的方式

2）隐式转换：如果是单独某个地方需要扩展类那就使用隐式转换
